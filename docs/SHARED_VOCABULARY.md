# Shared Vocabulary

Use these terms consistently across Prism repos and AI sessions.

## Core terms

| Term | Meaning |
|---|---|
| Beam | Canonical AI-facing reference, training, harvest, schema, contract, and anti-drift layer. |
| App repo | Independent repo that owns product runtime behaviour. |
| Context pack | Compact current-state file future AI sessions can read instead of rediscovering a repo. |
| Pattern card | Reusable, evidence-based description of a proven workflow or architecture pattern. |
| Harvest card | Curated external/open-source tool or idea with provenance, license, and adaptation notes. |
| Contract | Human-readable boundary agreement between apps or capabilities. |
| Schema | Machine-readable validation shape for packets, approvals, or metadata. |
| External write | Any action that publishes, sends, posts, pushes, or modifies a remote/public system. |
| Review-first | User sees and approves a proposed mutation before it changes app state or external systems. |
| Local-first | State and workflows should prefer local/user-controlled storage and avoid hidden cloud dependence. |
| Source-attached media | Media previews live beside the item they describe, not in detached folders/tabs unless a real asset library exists. |

## Avoid ambiguous words

- Avoid saying “secure” for a hosted admin route unless real platform auth is configured.
- Avoid saying “sent” for static contact forms that only open `mailto:`.
- Avoid saying “template” without distinguishing site/page templates from poster/export templates.
- Avoid saying “agent” when the implementation is only a reviewed assistant or script.
