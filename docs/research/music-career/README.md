## Key synthesis

- EPK is too narrow as the parent product concept. It should become an output/module of Prism Music/Career.
- Prism Music/Career needs an internal cockpit/visibility layer before public output/export/publishing.
- Dave's workflow must be authenticity-first and ND-aligned: batching, reuse, templates, review-first publishing, no fake persona.
- Spectra owns AI drafting/suggestion. Focus owns review-first task/session application. EPK/Music-Career owns public/professional source/output surfaces.
- Social, supporter, back-catalogue, and platform visibility are needed eventually, but canonical data ownership and safe read adapters are unresolved.

## Safety posture

No research finding overrides Beam's approval posture:

- AI suggestion = read-only at Spectra API boundary.
- Local draft = visible, discardable UI state.
- Local write = visible user action.
- Posting/sending/publishing = external-write requiring explicit approval.
- Deleting/overwriting canonical state = destructive and requires backup/rollback.

## Update rules

Refresh this index when Dave supplies newer music-career research, when app-side Music/Career implementation starts, when supporter/social/back-catalogue state ownership is decided, or when EPK is renamed/new repo is approved.
