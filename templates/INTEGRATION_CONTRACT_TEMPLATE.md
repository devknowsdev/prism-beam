# Integration Contract Template

## Title

Short contract name.

## Status

Draft, active, deprecated, or replaced.

## Source App

System that owns the source truth.

## Target App

System that consumes or reviews the data/action.

## Contract Owner

Beam owner, app owner, or team responsible for updates.

## Purpose

Why this boundary exists.

## What Crosses The Boundary

Data, references, drafts, approvals, or artifacts allowed through.

## What Must Not Cross The Boundary

State, secrets, hidden writes, private data, or actions that stay out.

## Data Shape / Schema Reference

Schema file or packet shape.

## Review And Approval Rule

Human review and approval expectations.

## Safe Read Path

Preferred read path and fallbacks.

## Safe Write Path

Allowed write path, if any.

## Failure/Rollback Notes

How to fail safely and recover.

## Validation

Checks future sessions should run.

## Future prompts can omit

Context this contract now stores.

## Update Rules

When this contract should be refreshed.
